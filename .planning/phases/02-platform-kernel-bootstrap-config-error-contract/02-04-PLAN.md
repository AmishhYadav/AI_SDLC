<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Inbound HTTP → CorrelationIdMiddleware | x-request-id / traceparent header accepted as traceId source |
| NestJS router → GlobalExceptionFilter | Unmatched routes produce NotFoundException caught by the filter |
| AppModule providers → APP_FILTER DI | Filter registration order determines which filter handles Prisma errors vs HTTP exceptions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Information Disclosure | Default NestJS 404 response (unmatched route) | mitigate | GlobalExceptionFilter with @Catch() catches NotFoundException thrown for unmatched routes; the standard envelope replaces NestJS's default { statusCode, message, error } shape which exposes framework version details |
| T-02-10 | Spoofing | x-request-id header accepted from untrusted caller | accept | Per D-01 design decision: traceId is correlation-only; no security decision (auth, access control, rate limit) is based on traceId value; the value is echoed in the error envelope as a debugging aid, not trusted |
| T-02-11 | Elevation of Privilege | setGlobalPrefix / enableVersioning ordering bug | mitigate | setGlobalPrefix('api') is called before enableVersioning() in main.ts; integration test verifies routes resolve under /api/v1/; acceptance criterion checks file order of the two calls |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | All packages verified by blocking human checkpoint in Plan 02-01 before any install ran |
</threat_model>
