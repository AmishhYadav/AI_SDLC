## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/client → Express (helmet + CORS) | Cross-origin requests and header injection cross here |
| Any IP → ThrottlerGuard | Rate-limiting applied before reaching handlers |
| HTTP body → ValidationPipe | Unknown/extra fields blocked before reaching handler |
| Non-prod server → Swagger endpoint | API schema exposure gated by NODE_ENV |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Elevation of Privilege | Helmet: clickjacking via missing X-Frame-Options, MIME-sniffing via missing X-Content-Type-Options | mitigate | app.use(helmet()) sets X-Frame-Options: SAMEORIGIN, X-Content-Type-Options: nosniff, HSTS, CSP, and 25+ additional headers; verified in integration test Test A/B |
| T-03-08 | Information Disclosure | CORS: cross-origin data exfiltration via missing origin allowlist | mitigate | app.enableCors({ origin: CORS_ORIGINS parsed from env }) — rejects origins not in the allowlist; CORS_ORIGINS is required Zod field (fails fast if missing); verified in integration test Test F |
| T-03-10 | Denial of Service | ThrottlerGuard: request flooding | mitigate | ThrottlerGuard registered as APP_GUARD with default 100 req/min/IP from THROTTLER_LIMIT env; overridable per-route with @Throttle(); TTL uses seconds() helper (not raw ms); in-memory storage (single-instance, Redis-ready) |
| T-03-05 | Tampering | ValidationPipe: mass-assignment via unknown body fields | mitigate | APP_PIPE ValidationPipe with whitelist: true + forbidNonWhitelisted: true; unknown fields cause 400; verified in integration test Test E |
