<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stub authentication to authorization | AUTH_MODE=stub must authenticate a principal but grant zero permissions |
| test fixtures to seeded data | Tests must not corrupt or depend on mutating the seeded permissions/roles |
| CI signal to reviewers | A green CI run must mean the real-DB RBAC proof actually executed, not that it silently skipped |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-12 | Elevation of Privilege | stub backdoor | mitigate | Explicit test: stub principal with no seeded UserRole gets 403 — proves stub never grants permissions (D-09) |
| T-05-13 | Elevation of Privilege | authN/authZ conflation | mitigate | Explicit test: missing token returns 401, valid-but-unauthorized returns 403 — proves the chain distinguishes the two (RBAC-03/RBAC-04) |
| T-05-14 | Information Disclosure | 403 body over the wire | mitigate | Deny-case assertion verifies the response message does not contain the missing permission code (D-54) |
| T-05-15 | Tampering | test fixture cleanup | accept | `afterAll` deletes only fixture rows; seeded reference data is read-only in tests; low risk in an ephemeral CI DB |
| T-05-16 | Repudiation / false confidence | silent test skip in CI | mitigate | `RBAC_REALDB_REQUIRED=1` (set job-wide in CI) makes a mis-wired/absent `DATABASE_URL` FAIL a non-skippable guard test instead of letting the whole real-DB RBAC proof skip to green (D-09) |

No package installs occur in this plan (Postgres runs as a CI service image, not an npm dependency) — no supply-chain checkpoint required.
</threat_model>
