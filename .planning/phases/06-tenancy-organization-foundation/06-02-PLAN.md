    @Injectable() class TenantGuard implements CanActivate. Constructor: Reflector (private readonly reflector), PrismaService (private readonly prisma — RAW, NOT TenantedPrismaService), ClsService (private readonly cls).

    canActivate(context: ExecutionContext): Promise<boolean> decision tree:

    Step 1 — @Public() bypass: reflector.getAllAndOverride<boolean>(IS_PUBLIC_KEY, [context.getHandler(), context.getClass()]); if true return true. (@Public() routes have no principal to validate org membership against.)

    Step 2 — @NoTenantScope() bypass: reflector.getAllAndOverride<boolean>(IS_NO_TENANT_SCOPE_KEY, [context.getHandler(), context.getClass()]); if true return true. (Cross-org routes: create org, list my orgs, explicit cross-org reads.)

    Step 3 — Fail-closed on missing principal: const request = context.switchToHttp().getRequest<{ headers: Record<string, string | string[] | undefined>; user?: CurrentUser }>(); if (!request.user) throw new ForbiddenException({ errorCode: TENANT_ERROR_CODES.NO_ORG_CONTEXT, message: 'Authentication required for tenant-scoped access.' }). (JwtAuthGuard runs before TenantGuard, so missing user = wrong guard order or unauthenticated.)

    Step 4 — Header extraction: const rawHeader = request.headers['x-organization-id']; const organizationId = Array.isArray(rawHeader) ? rawHeader[0] : rawHeader; if (!organizationId) throw new ForbiddenException({ errorCode: TENANT_ERROR_CODES.MISSING_ORG_HEADER, message: 'X-Organization-Id header is required.' });

    Step 5 — ACTIVE membership validation (using RAW PrismaService — NOT TenantedPrismaService): const member = await this.prisma.organizationMember.findFirst({ where: { organizationId, status: 'ACTIVE', deletedAt: null, user: { email: request.user.email, deletedAt: null } }, select: { id: true, userId: true } }); if (!member) throw new ForbiddenException({ errorCode: TENANT_ERROR_CODES.ORG_ACCESS_DENIED, message: 'Access denied. Verify your organization membership.' }). (D-02: same code for "org not found" and "not a member" — no org-existence oracle.)

    Step 6 — Populate CLS atomically (all three keys in sequence, no early return between them): this.cls.set('organizationId', organizationId); this.cls.set('organizationMemberId', member.id); this.cls.set('userId', member.userId); return true.

    CRITICAL: Do NOT inject or use TenantedPrismaService here. TenantGuard is the entity that POPULATES the CLS context; using the scoped client (which reads from CLS) before population creates a circular deadlock. Raw PrismaService only.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → TenantGuard | X-Organization-Id header is untrusted input; guard must validate against real membership row |
| TenantGuard → CLS | Three CLS keys written only after successful membership validation; no partial population |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Spoofing | X-Organization-Id header IDOR | mitigate | TenantGuard validates real OrganizationMember row with status=ACTIVE before accepting any orgId. Forged header without real membership → TENANT.ORG_ACCESS_DENIED (D-02). |
| T-06-06 | Information Disclosure | Org-existence oracle via error codes | mitigate | Same TENANT.ORG_ACCESS_DENIED for both "org not found" and "user not a member" — no differential timing or messages. Generic non-sensitive message. |
| T-06-07 | Tampering | Guard-order regression | mitigate | TenantGuard uses raw PrismaService; failing on missing request.user provides an early signal if guard order regresses (TenantGuard runs before JwtAuthGuard would cause all requests to fail with NO_ORG_CONTEXT). Integration test in plan 06-04 proves 401 before 403 on missing auth. |
| T-06-08 | Tampering | Header injection via array values | mitigate | Array.isArray(rawHeader) ? rawHeader[0] : rawHeader — takes only the first value; unit test (a) in guard spec covers this. |
| T-06-09 | Tampering | Bootstrap deadlock (scoped client in guard) | mitigate | TenantGuard constructor injects PrismaService only (raw). TenantedPrismaService is NOT in the guard's constructor — enforced by the module design and verified by the unit test (no TenantedPrismaService mock needed). |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages installed this plan. |
</threat_model>
